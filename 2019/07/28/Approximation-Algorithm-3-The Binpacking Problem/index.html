Week 3: The Binpacking Problem

Problem Definition:

本周讲的问题和上周背包问题有点相似，上周的是给出物品的体积和价值，要装到一个背包里使得价值之和尽可能大。本周的问题是$N$件物品，每件物品有一个体积$\{s_i\}$, 不失一般性地，可以约定$\{s_i\} \leq 1$, 有容量为$1$箱子, 要用尽可能少的箱子容纳所有的物品。

课程内容：

  • Next Fit Algorithm:
    • Algorithm: 依次考虑每个物品, 如果这个物品能放入当前箱子, 就放入, 否则关闭当前箱子, 打开一个新的箱子放入这个物品。
    • Approximation Ratio: 把编号相邻的箱子配对, 考虑箱子$2i-1$和箱子$2i$。我们打开了箱子$2i$, 说明箱子$2i-1$里面的物品体积之和加上箱子$2i$里面的第一个物品的体积, 必然是大于$1$的，否则按照算法流程会把那个物品放入箱子$2i-1$. 假设按照算法流程用了$K$个箱子, 装入最后一个箱子的第一个物品编号是$L$. 那么有$OPT \geq \sum\limits_{i=1}^{N}{s_i} \gt \sum\limits_{i=1}^{L-1}{s_i} \gt \lfloor \frac{K}{2} \rfloor \geq \frac{K-1}{2}$, 即$K \lt 2OPT+1$, 也就是$K \leq 2OPT$.
    • Tightness: 课程没有给出一个具体的例子, 我google到了一份资料, 里面给出了一个$2OPT-2$的例子: 一共$4n$个物品, 满足$s_{2i-1} = \frac{1}{2}, s_{2i} = \frac{1}{2n}$. 显然$OPT = n+1$, 而Next Fit给出的结果是$2n$.
    • Special Case: 所有物品的$s_i \lt \frac{1}{3}$.
      • Analysis: 当关闭一个箱子的时候(除了最后一个箱子), 这个箱子里面物品体积之和必然$\geq \frac{2}{3}$, 否则还可以接着装物品。 假设用了$K$个箱子, 第一个装入第$K$个的物品编号是$L$, 那么有$OPT \gt \sum\limits_{i=1}^{N}{s_i} \gt \sum\limits_{i=1}^{L-1}{s_i} \geq \frac{2}{3}(K-1)$. 因此有$K \leq \frac{3}{2}OPT$。
      • 相应地，可以推导所有物品的$s_i \lt \epsilon$的时候, 有$K \leq \frac{1}{1-\epsilon}OPT$。
  • A very special case: 物品体积较大, 不同的size比较少(large items, few distinct sizes).
    • New Definitions:
      • Configuration: 指将若干物品装入一个箱子的某种方法。比如往一个箱子中装入2个size为0.2的物品和1个size为0.6的物品。
      • $A_{c, s}$: configuration $c$中有$A_{c, s}$个size为$s$的物品。
    • Observation: 假设物品体积的下界是$\epsilon$, 一共有$K$种不同的size, 那么不同configuration的数量$\leq K^{\frac{1}{\epsilon}}$.
    • Linear Programming:
      对每种size $s$, 设有$n_s$个物品, 那么必须有$\sum\limits_{c}{A_{c, s}x_c} \geq n_s$.
      目标函数就是$\sum\limits_{c}{x_c}$
    • Rounding: 求出实数线性规划的解向上取整。
    • Analysis: 显然求出的解是合法解，而$OPT \geq \sum\limits_{c}{x_c} \geq \sum\limits_{c}{(\lceil x_c \rceil-1)}$, 因此我们求出的解不会比$OPT$多于线性规划变量的个数, 也就是configuration的数目。而configuration的数目$\leq K^{\frac{1}{D}}$。 综上近似解$\leq OPT + K^{\frac{1}{D}}$。
  • A less special case: 物品体积较大, 不同的size比较多(large items, many distinct sizes)。
    • Reduce to previous case: 考虑Rounding， 将物品按size排序， 然后分成若干组，将每组物品的size round成最大的那个。
    • Adaptive Rounding:
      • 假设$\min s_i \geq \epsilon$.
      • 将物品按照size从小到大排序。
      • 将物品分组，每组的数目是$G = \lfloor N\epsilon^{2}\rfloor$(除了最后最后一组)。
      • 将每组内物品的size全部变成组内最大的。
      • 通过上面LP的方法求出最优解.
    • Analysis(有些tricky):
      • 显然上面求出的解是合法的，因为将物品的体积变大了能装下，原问题更加可以装下。
      • 引入一种新的rounding：不是将每组内物品的size全部变成组内最大的, 而是变成上一组物品的最大size(第一组就全变成0)。为了表示方便，把第一种rounding后的问题表示为$U(up, 向上变大)$, 第二种rounding后的问题表示为$D(down, 向下变大)$.
      • 考虑$D$和$U$的区别, 假设有$K$组物品, $D$的第$2$组物品和$U$的第$1$组物品是一样的, $D$的第$3$组物品和$U$的第$2$组物品是一样的$\cdots$$D$的第$i+1$组物品和$U$的第$i$组物品是一样的($1 \leq i \leq K-2$)。如下图所示： 而$D$的第$1$组物品体积全是$0$, 可以忽略掉。因此$D$和$U$的区别就是$D$的最后一组和$U$的第$K-1, K$组. 而$D$的最后一组其实完全包含在$U$的第$K-1$组里。 综上, 如果忽略$D$的第$1$, 那么有$card(U-D) \leq \lfloor N\epsilon^{2}\rfloor$,从而容易得出$OPT(U) - OPT(D) \leq \lfloor N\epsilon^{2}\rfloor$.
      • 设$O$是原问题, 那么有$OPT(D) \leq OPT(O) \leq OPT(U)$.
      • 结合上2条, 可以推出$OPT(U) \leq OPT(O) + \lfloor N\epsilon^{2}\rfloor$.
      • 因为$\min s_i \geq \epsilon$, 所以有$OPT(O) \geq \sum\limits_{i=1}^{N}{s_i} \geq N\epsilon$.
      • 进一步推出$OPT(U) \leq OPT(O) + \lfloor N\epsilon^{2}\rfloor \leq OPT(O) + \epsilon OPT(O) = (1+\epsilon)OPT(O)$.
      • 综上, Adaptive Rounding的近似比是$1+\epsilon$.
  • General Cases: 去掉物品体积较大的限制。
    • Algorithm:
      • 将体积$\geq \epsilon$的物品, 按照上面的方法做一遍。
      • 剩下的物品按照现有的箱子有空就塞，没空就开新箱子的原则贪心。
    • Analysis:
      • 假设没有开新的箱子，那么有:
        $output(All) = output(Large) \leq (1+\epsilon)OPT(Large) \leq (1+\epsilon)OPT(All)$.
      • 假设开了新的箱子, 那么除了最后一个箱子, 其他箱子里面装的物品体积之和肯定$\gt 1-\epsilon$。那么有$OPT(All) \gt (1-\epsilon) (output(All)-1)$, 即$output(All) \leq \frac{1}{1-\epsilon}OPT(All)$.
      • $\frac{1}{1-\epsilon} = 1+\epsilon+\epsilon^2+\epsilon^3\cdots$, 所以$\frac{1}{1-\epsilon} \geq 1+\epsilon$. 综合一下就是$output(All) \leq \frac{1}{1-\epsilon} OPT(All)$.
  • 前沿科技：Narendra Karmarkar和Richard Karp提出了$OPT+O(log^2(OPT))$的算法, Rebecca Hoberg和Thomas Rothvoss改进了算法，做到了$OPT+O(log(OPT))$。

如何优雅地使用 Python

2020-05-23 Knowledge Learning 阅读量

导语

每当我在用 python 做一些 task 的时候，对这门语言总有强烈的距离感。

  • 很多基础的函数不会调用，要现场百度起来。
  • 更糟糕的是，没有掌握 Python 灵巧的编程思想，还是一味用 C++ 的思路去写代码。

于是我就开了这篇博客，记录一些反复用到的技巧和方法，培养 Python 的思维习惯。

常用函数

  • any(<list>)，all(<list>)，sum(<list>)

  • 用 enumerate() 来带序号遍历 list

  • 取 dict 里的元素最大值：max(vote, key = vote.get)

  • 字符串相关的函数

  • 正则表达式

  • Lambda 函数

    • 注意：lambda 函数会绑定全局变量地址而非其值。以下代码的解决方案 见此。

美化代码

  • itertools 库里有很多实用的枚举姿势

  • namedtuple 具名元组：在元组的基础上，支持用属性名去访问元素。

  • partial：灵活设置函数的默认参数，使其变成新函数。

第三方库推荐

  • pyparsing：可以用来定制小型的解析器，比起枯燥死板的正则的语法，更加贴近思维习惯。

  • tesserocr & pytesseract：对给定图片进行文本检测，并返回识别后的字符串。

    • 不太适合很小的图（比如只包括一个字符），进行适当的缩放和重复会有更好的识别效果。

  • Sympy：一个很厉害的数学表达式运算的库！一个比较详细的中文介绍

常见的私有方法

类或函数的方法 效果
__doc__ 描述信息，即定义开头长注释里的字符串；可用于模块、函数、类。
__new__，__init__ 前者用于类对象的创建（静态方法），后者用于对象的初始化（实例方法）。前者必须要给出返回值（对象），作为后者的第一个参数。
__del__ 类的析构函数
__module__，__class__ 前者表示对象所在模块，后者表示对象所属的类
__dict__ 类或对象中的所有成员（dict 形式）
__str__ 类需要转为字符串时（比如打印该对象）触发，类似于 Java 的 toString
__call__ 当对象后面接 () 后会触发
__getitem__ 当对象后面接 [] 后会触发
__setitem__，__delitem__ 连同上面的 __getitem__，使类像 dict 一样使用
__iter__， __next__ 使该类变得可迭代，可以用 for ... in ... 遍历

Jupyter Notebook

  • 优点：即写即用，一直缓存结果，适合开发。
  • 缺点：不支持代码补全和跳转；需要额外开一个命令行。
  • 把 Block 从代码转为标记后，可以写 markdown 源代码（”运行“后显示效果）。
  • 展示图片
    • 用 cv2.imshow("WindowsName", a) 无法正常显示图片，需要再后面加 cv2.waitKey(delay)，如 delay=0 则跳出一张图片（鼠标关闭后才会继续运行）。
    • 用 plt.imshow(a) 可以画出 a，但是会在代码运行完后再画。
    • 用 plt.imshow(a); plt.show() 后可以立即输出图片（也可以连续画出多张图片）。

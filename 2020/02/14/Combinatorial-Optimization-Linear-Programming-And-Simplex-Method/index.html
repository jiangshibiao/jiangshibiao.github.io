单纯形法

个人感觉线性规划的求解都比较繁琐，不感兴趣的可以先不看, 实际应用也很少真的去直接求解，而是用组合的方法。目前已经有多项式的算法可以解决线性规划，而单纯形最坏时间是指数级的。不过，在大多数情况下，它的运行效率很高，甚至比多项式算法要快。

一些约定

  1. 线性规划有可行解. (无可行解的情况可以用书中CH2.8的方法判断, 见下文)
  2. 矩阵$A$的规模是$m \times n$, 且$rank(A)=m$, 即行满秩. (不满秩的情况会在书中CH2.8涉及, 见下文)
  3. $b_i \geq 0$. (如果不满足，只要第$i$行乘$-1$)
  4. 最优解不是无穷小. (下文会介绍怎么判断会不会是无穷小)

基本可行解

基本可行解$x$的定义如下:

  1. 取$A$线性无关的$m列$, 记为方阵$B$, 对应的列标集合记为$\mathcal{B}$.
  2. $x_j = 0(j \notin \mathcal{B})$.
  3. $x_{\mathcal{B}} = B^{-1}b$.

通俗点说就是取$A$线性无关的$m列$, 让剩余列对应的$x_j=0$, 就变成了解方程$Bx_{\mathcal{B}} = b$.
容易证明, 在上面的约定1,2下, 基本可行解一定存在。(证明见书Theorem 2.1, 另外CH2.8的方法其实暗含了基本可行解一定存在)
为什么要基本可行解这个概念呢？因为一定有一个基本可行解就是最优解。这个证明就比较复杂了, 不感兴趣可以略过。

  1. 首先容易证明线性规划对应的可行解区域是一个convex polytope。
  2. 基本可行解和polytope的顶点一一对应。(证明见书Theorem 2.4)
  3. polytope的任意点是其顶点的凸组合(书Theorem 2.3提到, 不过也没有证明, 直接引用的别人的结论)
  4. 由1,2,3容易推出某个基本可行解就是最优解。假设不是，即polytope的某个内点是最优解, 那么考虑把它表示成顶点的凸组合，凸组合里一定有一个顶点比它优。

由此我们得到一个很暴力的算法：枚举所有的基本可行解, 最多$\binom{n}{m}$种可能。
单纯形法稍微取巧有点，本质上是基于local search, 从polytope的某个顶点出发，不断走到相邻的一个更优的顶点。 最坏情况下，仍然需要计算$\binom{n}{m}$次基本可行解(例子见书CH8.6)。

基本可行解的切换

假设当前基本可行解是$\mathcal{B}$, 定义它的neibourhood:$\{\mathcal{B’} \mid \mathcal{B’} = \mathcal{B} \backslash i \cup j, i \in \mathcal{B}, j \notin \mathcal{B}\}$.
